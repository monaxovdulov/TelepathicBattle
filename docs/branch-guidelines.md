# Спецификация для названий веток:

#### Основные правила:
1. Используйте **маленькие буквы** и **дефисы (-)** для разделения слов.
2. Ветки должны отражать цель работы: новую функциональность, исправление ошибок или экспериментальную работу.

#### Типы веток:
1. **Основные ветки**:
   - `main` — основная стабильная ветка с релизами.
   - `dev` — ветка для общей разработки, куда сливаются все изменения перед проверкой и слиянием в `main`.

2. **Функциональные ветки**:
   - **feature/название-функции** — для работы над новыми функциями.
     - Пример: `feature/ai-logic`, `feature/player-interface`
   - **bugfix/название-исправления** — для исправления багов.
     - Пример: `bugfix/input-validation`, `bugfix/display-error`
   - **hotfix/название-срочного-исправления** — для срочных исправлений багов в ветке `main`.
     - Пример: `hotfix/login-issue`

3. **Экспериментальные или тестовые ветки**:
   - **experiment/название-эксперимента** — для экспериментов и тестов.
     - Пример: `experiment/new-algorithm`

#### Пример структуры веток:
- `main` — основная ветка.
- `dev` — ветка для разработки.
- `feature/ai-logic` — работа над логикой компьютера.
- `bugfix/input-validation` — исправление ошибок ввода.
- `experiment/alternative-ai` — тестирование нового алгоритма ИИ.


### Резюме:
- Названия веток должны быть понятными и отражать цель работы: `feature/`, `bugfix/`, `hotfix/`.
- Придерживайтесь общей структуры, чтобы всем было понятно, над чем идёт работа.
